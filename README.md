# word-chain-game-with-ai-cli

A CLI word chain game — take turns with an AI opponent, each word starting with the last letter of the one before it (apple → elephant → tiger → rat → ...).

| Area | Technology |
|---|---|
| Language | Python 3.9+ |
| CLI framework | click |
| AI opponent | Gemini API, with automatic offline fallback if no API key is set |
| Storage | Local JSON file (repository pattern) |

## AI opponent

- **With `GEMINI_API_KEY` set**: the AI's moves are generated by Gemini (model fallback chain: `gemini-2.0-flash` → `gemini-1.5-flash` → `gemini-2.5-flash`, with retry on 503). Every word Gemini returns is still validated locally (correct starting letter, real word, not reused) before being accepted.
- **Without a key**: the AI picks randomly from a bundled ~130k-word English dictionary (`word_chain/data/words.txt`) — the game is fully playable offline, no setup required.
- If Gemini fails mid-game (network issue, bad response) it silently falls back to the offline picker for that turn rather than ending the game, and the CLI honestly labels each AI move with 🤖 (Gemini) or 🎲 (offline) so you always know which one actually produced it.

AI client code: [`word_chain/ai_client.py`](word_chain/ai_client.py).

## Setup

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install --upgrade pip
pip install -e ".[dev]"
```

Optional — enable the real AI opponent by copying `.env.example` to `.env` and adding a free [Gemini API key](https://aistudio.google.com/apikey):
```bash
cp .env.example .env
```

## Commands

```bash
wordchain play       # start an interactive game (type 'quit' to end early)
wordchain history     # list past games
pytest                # run the test suite
```

## Architecture

Same repository-pattern layering as this project's siblings, [expense-tracker-cli](../expense-tracker-cli) (TypeScript) and [bookmark-manager-cli](../bookmark-manager-cli) (Python) — plus a second swappable abstraction for the AI opponent itself:

- `word_chain/models.py` — `GameResult` dataclass
- `word_chain/words.py` — loads the bundled word list, validates words, picks random offline moves
- `word_chain/repository.py` — `GameRepository` ABC + `JsonGameRepository` (local JSON storage for game history)
- `word_chain/ai_client.py` — `AiClient` ABC + `GeminiAiClient` (real) + `OfflineAiClient` (fallback/no-key mode)
- `word_chain/service.py` — game rules (word validation, turn orchestration), depends only on the repository and AI-client interfaces
- `word_chain/cli.py` — click CLI wiring

Game history lives in `data/history.json` (gitignored).
